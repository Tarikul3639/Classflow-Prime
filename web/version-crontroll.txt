Step 1: ------------------Install Packages (Inside web folder)----------------

# Navigate to web directory
cd web

# Install semantic-release and its plugins locally
npm install --save-dev semantic-release semantic-release-monorepo @semantic-release/changelog @semantic-release/git @semantic-release/github @semantic-release/npm

Step 2: ----------------Create Configuration File--------------------------------

Create a file named .releaserc.json inside the web folder.
File Path: web/.releaserc.json

{
  "extends": "semantic-release-monorepo",
  "tagFormat": "web-v${version}",
  "plugins": [
    "@semantic-release/commit-analyzer",
    "@semantic-release/release-notes-generator",
    "@semantic-release/changelog",
    "@semantic-release/npm",
    [
      "@semantic-release/git",
      {
        "assets": ["package.json", "CHANGELOG.md"],
        "message": "chore(web): release ${nextRelease.version} [skip ci]\n\n${nextRelease.notes}"
      }
    ],
    "@semantic-release/github"
  ]
}

Step 3: -------------------Create GitHub Action Workflow---------------------

Now, go to your project root and create the workflow file. This file tells GitHub to run the release process only when files in the web folder are changed.

File Path: .github/workflows/web-release.yml

name: Web Release
on:
  push:
    branches:
      - main
    paths:
      - 'web/**' # Trigger only if files in the web folder change

jobs:
  release:
    runs-on: ubuntu-latest
    permissions:
      contents: write
      issues: write
      pull-requests: write
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 'lts/*'

      - name: Install & Release Web
        working-directory: ./web
        run: |
          npm install
          npx semantic-release -e .releaserc.json
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          NPM_TOKEN: ${{ secrets.NPM_TOKEN }}


Step 4: ---------------------------Enable Permissions in GitHub---------------
To allow the bot to create releases and push the CHANGELOG.md back to your repo:

Go to your GitHub Repository -> Settings.

Click Actions > General.

Under Workflow permissions, select "Read and write permissions".

Click Save.

Step 5:--------------------------- How to trigger the release-------------------
From now on, use Conventional Commits to tell the system how to bump the version.

# To trigger a Minor Version (e.g., 1.0.0 -> 1.1.0)
git add .
git commit -m "feat(web): add new login page"
git push origin main

# To trigger a Patch Version (e.g., 1.0.0 -> 1.0.1)
git add .
git commit -m "fix(web): fix navigation bug"
git push origin main
